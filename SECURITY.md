# Reporting Security Issues

We take all security reports seriously. If you believe you have found a security
issue in this project, **please do not create a public GitHub issue.**

Instead, report the issue privately through the AWS vulnerability disclosure
process: <https://aws.amazon.com/security/vulnerability-reporting/>

Please include the following information when reporting:

- A description of the issue and its potential impact
- Steps to reproduce, including any specific configuration required
- Any proof-of-concept code or commands
- Suggested mitigations, if known

We will acknowledge receipt within a reasonable time and provide an estimated
timeline for response and remediation.

This sample is published under MIT-0 and is not a managed AWS service. The
intended consumers are AWS customers operating it inside their own accounts.
Reports concerning the design of the sample (rather than a managed AWS service)
will be triaged on the same channel and patched in this repository.
